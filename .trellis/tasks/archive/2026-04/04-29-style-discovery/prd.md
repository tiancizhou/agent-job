# feature: 提升预设风格发现性

## Goal

在不增加用户发送摩擦的前提下，让用户更容易发现并使用 QuickDa 预设的提示词风格，提高生成结果的视觉稳定性和趣味性。

## What I already know

* 用户发现问题：很多用户不知道可以使用预设提示词风格。
* 产品定位应保持 C 端轻量、好玩、有趣，不应增加强制选择弹窗。
* 当前首页 Hero 输入区和聊天输入区都有风格入口，但主要表现为一个调色盘图标。
* 当前风格弹层展示「无风格」和后台激活的风格卡片。
* 后台风格管理已有名称、提示词、排序、激活状态。
* 当前设备偏好任务也在修改 `ChatPanel.vue`，本任务需要避免冲突，继续基于同一输入区工具栏扩展。

## Requirements

* 首页 Hero 输入区要让预设风格入口更明显。
* 聊天输入区要让当前风格状态更容易被看到。
* 默认仍为无风格，不强制选择、不阻塞发送。
* 用户可以在发送前快速理解「可选择风格」并打开风格选择。
* 选择风格后应继续沿用现有 `selectedStyleId` / `setAppStyle()` 流程。
* 仅使用已有后台激活风格，不新增数据库字段或新推荐算法。

## Acceptance Criteria

* [x] 首页输入区显示可见的风格提示/胶囊，而不是只靠隐藏图标。
* [x] 聊天输入区显示当前风格名称或清晰的「选风格」入口。
* [x] 用户仍可选择「无风格」。
* [x] 不选择时仍可直接发送。
* [x] 已选风格在首页创建应用和已有应用修改中继续生效。
* [x] `npm --prefix frontend run build` passes。

## Definition of Done

* 前端构建通过。
* 变更范围保持在风格可发现性 UI 和必要的本地状态/样式。
* 不引入新的后端字段、接口或强制选择流程。

## Out of Scope

* 不做强制弹窗。
* 不做 LLM 自动推荐风格。
* 不做风格使用埋点统计。
* 不做风格持久化偏好。
* 不修改后台风格管理数据结构。

## Technical Approach

* 在首页 composer footer 内将风格入口从单图标增强为「风格：当前/选风格」胶囊，可点击打开现有风格弹层。
* 在聊天 toolbar 内显示当前风格短标签，未选择时显示「选风格」。
* 保留现有风格弹层和 `toggleHomeStyle()` / `selectChatStyle()` 行为。
* 优先用现有 BEM scoped CSS 扩展，不新增组件或依赖。

## Decision (ADR-lite)

**Context**: 风格功能存在但入口不明显，用户不知道可以使用预设风格。

**Decision**: 采用轻量可见胶囊入口，而不是弹窗或复杂推荐系统。

**Consequences**: 发现率提升且发送摩擦低；暂不解决自动匹配最佳风格的问题，后续可基于使用情况再做推荐。
